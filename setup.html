Встановлення Git

Для того, щоб почати працювати з системою контролю версій Git її необхідно попередньо встановити. В інструкції описаний процес завантаження, установки системи контролю версій Git на сервера з операційною системою Windows і підключення до віддаленого сховища GitHub.

Завантаження та інсталяція

Завантажте інсталятор з офіційного сайту. Завантаження почнеться автоматично.

Перейдіть до Завантажень і запустіть завантажений файл.

Вкажіть шлях до каталогу в який буде встановлений Git.

Оберіть всі необхідні для вас функціїї

Введіть ім'я директорії, яка буде створена в Start Menu. При необхідності можна змінити шлях за допомогою кнопки Browse.

Оберіть спосіб використання з командного рядка:

  • Use Git from Git Bash only - використання тільки з командного рядка Bash.
  • Use Git from the Windows Command Prompt - використання командного рядка Bash, а також мінімальний набір команд Git з консолі Windows.
  • Use Git and optional Unix tools from the Windows Command Prompt - використання Git і утиліт Unix з командного рядка Windows, в цьому випадку будуть перезаписані деякі утиліти Windows, наприклад find і sort.

Оберіть бібліотеку, яка буде використана при підключенні по протоколу HTTPS:

  • OpenSSL - сертифікати сервера будуть перевірятися з використанням Unix-файлу ca-bundle.crt.
  • Windows Secure Channel - сертифікати сервера будуть перевірятися з використанням стандартної бібліотеки Windows.

Переконайтеся, що ви вибрали спосіб обробки закінчення рядків «Checkout Windows-style, commit Unix-style line endings». Це значення гарантує, що Git перетворює LF в CRLF при перевірці текстових файлів. При виконанні текстових файлів CRLF також перетворюється в LF. Це міра сумісності для захисту нових рядків в текстових файлах, що дозволяє легко працювати з текстовими файлами в Windows і на платформах Unix. Примітка: LF і CRLF - керуючий символ для розриву рядків в Unix і Windows відповідно.

Далі необхідно конфігурувати використовуваний термінал:

  • MinTTY - термінал Unix;
  • Windows - стандартний термінал Windows.

Відзначте галочками потрібні вам додаткові функції:

  • File system caching - кешування файлової системи.
  • Git Credential Manager - включити менеджер облікових даних.
  • Symbolic links - дозволити символічні посилання.

Налаштування Git

У Git є налаштування користувача, від якого буде йти робота. Це розумна і необхідна річ, так як коли створюється Комміт, гіт бере саме цю інформацію для поля Author.

Щоб налаштувати ім'я користувача та пароль для всіх проектів, потрібно прописати наступні команди:

git config --global user.name ”Ivan Ivanov”
git config --global user.email ivan.ivanov@gmail.com

Якщо є необхідність для конкретного проекту змінити автора (для особистого проекту, наприклад), можна прибрати --global, і вийде так :

git config user.name ”Ivan Ivanov”
git config user.email ivan.ivanov@gmail.com
